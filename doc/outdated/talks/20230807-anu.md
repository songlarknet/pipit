## Example: thermometer driver
```
driver             sensor
  |                   |
  |        read       |
  |------------------>|
  |  ok, 25°C, fresh  |
  |<------------------|
  |                   |
  |        read       |

  |<------------------|
  |                   |
  |                   |
  |                   |↩ dead
  |                   |
  |                   |
  |        read       |
  |------------------>|
  |         nok       |
  |<------------------|
  |                   |
  |       reset       |
  |------------------>|
  |                   |
  | enable interrupts |
  |------------------>|
  |                   |
  |        read       |
  |------------------>|
  |   ok, 0°C, stale  |
  |<------------------|
  |                   |
  |                   |
  |                   |
  |                   |
  |        read       |
  |------------------>|
  |  ok, 26°C, fresh  |
  |<------------------|
```

Property we want:
  ◇ □ stable => ◇ fresh

Problems:
 ◇ □ stable is too strong an assumption, we don't care about stability a year from now
 ◇ fresh is too weak: we want it to be stable today
 not computable: computations cannot see future

Past, bounded:
  ■_n stable => ♦_n fresh


```
node driver(
  rsp: TRSP;
) returns (
  cmd: TCMD;
  temp_read_success: bool;
  temp_fresh:   bool;
  temp: TEMPERATURE;
)
  let cmd = rec cmd.
    let therm_reset_ok = rsp.cmd_ok in
    let therm_init_ok = false ->
      if not therm_reset_ok then false
      else if pre cmd = SET_INT_ENABLE and rsp.cmd_ok then true
      else pre therm_init_ok
    in
      if not therm_reset_ok then RESET
      else if not therm_init_ok then SET_INT_ENABLE
      else READ
  in

  let temp_read_success = false -> pre cmd = READ and rsp.cmd_ok and rsp.fresh in
  let temp_fresh = anyn(temp_read_success, FRESH_LIMIT) in
  let temp =
      0 ->
      if temp_read_success then rsp.temp
      else pre temp
  in
    (cmd, temp_fresh, temp)
```


## Proving properties

```fs
driver : stream response -> stream (command & temp & bool)
driver rsp =
  ...
  let temp_read_success =
    (fby NONE cmd) = READ and rsp.cmd_ok and rsp.fresh in
  let temp_fresh = anyn(temp_read_success, FRESH_LIMIT) in
  let temp =
    if temp_read_success
    then rsp.temp
    else (fby 0 temp)
  in
  (cmd, temp, temp_fresh)
```

## Generating executable code

```
[@@(Tac.postprocess_with XL.tac_extract)]
let state = XX.state_of_exp driver'

[@@(Tac.postprocess_with XL.tac_extract)]
let reset = XL.mk_reset driver'

[@@(Tac.postprocess_with XL.tac_extract)]
let step (rsp: response) =
  XL.mk_step driver' (rsp, ())
```

```
let tac_extract () =
  Tac.norm [nbe; delta; primops; iota; zeta];
  Tac.trefl ()
```

```
inline_for_extraction
let mk_step (#input #state #result: Type) (t: X.exec input state result) (inp: input) (stref: B.pointer state) : ST result
    (requires (fun h -> B.live h stref))
    (ensures (fun h _ h' -> B.live h' stref)) =
  let st  = !*stref in
  let res = t.eval inp st in
  let st' = t.update inp st in
  stref *= st';
  res
```

The current translation is to a pure transition system. It's relatively easy to normalize away the translation. Will it be harder to normalize the translation once this is generating mutable code?

## Trusted computing base

TCB comparison:

Code generation:
* Vélus, verified Lustre compiler:
  Coq, OCaml extraction of Vélus and CompCert, CompCert assembly semantics, linker, OCaml runtime
* Heptagon, open-source (unverified) Lustre compiler
  Heptagon implementation, OCaml runtime, C compiler, etc
* SCADE, proprietary *certified* compiler
  certified KCG backend

* Pipit:
  F*, OCaml runtime, Karamel (F*->C) compiler

Proofs:
* Kind2:
  Z3, Kind2 semantics, etc
* Pipit:
  Z3, F* semantics, normaliser, etc
